Privacy Policy — dailyhoopstats uploader

Effective date: July 21, 2026

1. Use of YouTube API Services

dailyhoopstats uploader uses YouTube API Services (YouTube Data API v3). By using this application, the user agrees to be bound by the YouTube Terms of Service. Google's handling of information is described in the Google Privacy Policy.

1b. Use of the TikTok Content Posting API

The application also uses the TikTok Content Posting API to upload the developer's own videos to the developer's own TikTok account (@dailyhoopstats), subject to the TikTok Terms of Service and the TikTok Privacy Policy. The data practices described in this policy apply equally: the only data stored are the OAuth tokens issued by TikTok, kept locally; no data about any TikTok user is accessed, collected or shared.

2. Who uses this application

The application is a private desktop script operated solely by the developer, who is also the owner of the YouTube channel it uploads to. It has no other users and is not distributed to third parties.

3. What data the application accesses

  • Through Google OAuth 2.0 with the youtube.upload scope, it obtains an access credential authorizing uploads to the owner's channel.
  • It does not access, collect, store or share any personal data of any YouTube user, viewer or third party.
  • It does not use cookies, analytics, advertising identifiers, or any tracking technology.

4. What data the application stores

The only data stored are the OAuth 2.0 tokens issued by Google, kept in a local file on the developer's own computer. Tokens are never transmitted anywhere other than to Google's OAuth endpoints, and are never shared with third parties.

5. Data sharing

No data is shared with any third party. The only external service the application communicates with is the YouTube Data API itself.

6. Revoking access

Authorization can be revoked at any time from the Google security settings page: https://security.google.com/settings/security/permissions (also reachable at myaccount.google.com/permissions). For TikTok, authorization can be revoked in the TikTok app under Settings & privacy → Security & permissions → Apps and services. Revoking access immediately invalidates the stored tokens.

7. Data deletion

Stored authorization data can be deleted at any time by deleting the local token file. To request deletion of any data related to this application, contact juacuec2@gmail.com; requests are honored within 7 days.

8. Changes to this policy

Any material change to this policy will be published on this page with an updated effective date.
